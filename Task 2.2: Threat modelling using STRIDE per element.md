## 2.2: Threat Modelling using STRIDE per element

#### Element: Web application / Complaint form

| Interaction            | T                                                                                                                                            | I                                                                                                                                | D                                                                                                                                                        |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Submitting a complaint | Input fields could be manipulated with malicious input to change how the system behaves.<br><br>CIA: I<br><br>ATT&CK: T1190<br><br>CAPEC: 66 | Sensitive complaint data could be exposed if the form handling or error messages are not secure.<br><br>CIA: C<br><br>CAPEC: 118 | A large number of requests could overwhelm the complaint submission system and make it unavailable.<br><br>CIA: A<br><br>ATT&CK: T1499<br><br>CAPEC: 125 |

#### Element: Authentication System

| Interaction      | S                                                                                                                            | R                                                                                 | I                                                                                         | D                                                                                                        | E                                                                                                                            |
| ---------------- | ---------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| Logging into CMS | An attacker could use stolen credentials to log in as a legitimate user.<br><br>CIA: C<br><br>ATT&CK: T1110<br><br>CAPEC: 49 | A user could deny login activity if proper logging is not in place.<br><br>CIA: I | Error messages during login could reveal valid usernames.<br><br>CIA: C<br><br>CAPEC: 115 | Repeated login attempts could lock accounts or slow down the system.<br><br>CIA: A,<br><br>ATT&CK: T1110 | If credentials are compromised, an attacker could gain access to higher-level accounts.<br><br>CIA: C/I<br><br>ATT&CK: T1078 |

| ------------------------------ | --------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
